# Naming and Positioning

This document records the naming discussion and recommended external positioning.

## 1. Positioning

The project should not be positioned as merely:

- a cheap model switcher,
- a provider router,
- a prompt truncator,
- or a plain summarizer.

A better positioning is:

> **A local-first information compiler for hybrid LLM systems**

or

> **A self-routing, residual-preserving, reasoning-oriented information compiler**

## 2. Recommended names

Several names were proposed during the discussion.

### Strong candidates

#### Coretto
Recommended as the strongest product-style name.

Why it works:

- memorable,
- compact,
- product-like,
- evokes compression to the core.

Suggested line:

> **Coretto — Compile queries to their core.**

#### Kernel
Recommended as the strongest research/system-style name.

Why it works:

- aligns with the idea of the irreducible query core,
- fits the “minimal sufficient context” philosophy,
- strong theoretical flavor.

Suggested line:

> **Kernel — Minimal sufficient context for cloud reasoning.**

#### Prism
Recommended as the best “methodological image” name.

Why it works:

- evokes splitting complex queries into meaningful components,
- aligns with the idea of decomposition before compression,
- elegant and memorable.

Suggested line:

> **Prism — Refract complex queries into reasoning-ready cores.**

## 3. Shortlisted product slogans

### For Coretto

- Compile the core.
- Less prompt, more reasoning.
- From messy queries to minimal intelligence.

### For Kernel

- Find the irreducible query.
- Only the core goes to the cloud.
- Minimal context, maximal reasoning.

### For Prism

- Split the noise. Keep the signal.
- Refract queries into reasoning cores.
- Understand first. Send less.

## 4. Recommended current stance

If a public-facing working name is needed now:

- use **Coretto** for product-like external identity,
- use **KIR-Lite** as the internal representation name,
- keep architecture-neutral descriptions in technical docs.
